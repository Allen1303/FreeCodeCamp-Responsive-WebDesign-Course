Introduction

JavaScript (JS) is a versatile programming language primarily used to create dynamic and interactive elements on web pages. It's often referred to as a client-side scripting language because the code executes directly in the user's web browser, as opposed to server-side languages that run on web servers.

Here are some key characteristics of JavaScript:

  • Interpreted: JavaScript code doesn't need to be compiled into machine code before execution. Instead, the browser interprets and executes it line by line.
  • High-level: JavaScript allows developers to write code without getting into the nitty-gritty details of machine-level operations.
  • Dynamically Typed: JavaScript variables don't have a pre-defined data type. The type of data a variable holds can change during the program's execution.
  • Object-Oriented: JavaScript supports object-oriented programming concepts like objects, classes, inheritance, and more.

JavaScript and Java

While JavaScript shares some syntax with Java (like naming conventions and control flow), it's fundamentally different. JavaScript is dynamically typed (no pre-defined variable types) and uses prototypes for inheritance, unlike Java's statically typed classes. JavaScript offers more flexibility, allowing functions as object properties and omitting variable/function type declarations.

Variables

Imagine tiny boxes that hold data you can use throughout your program. That's what variables are in JavaScript!

What they are:

  • Variables are named containers that store information like numbers, text, or even more complex data. You give them a name (following specific rules) to easily refer to the stored data later.

Why they matter

  • Variables make code more readable and maintainable. Instead of repeating values, you use clear variable names
  • They allow your code to be flexible. You can change the data stored in a variable during program execution.

    • let name = "Alice"; // Variable named "name" stores the text "Alice" console.log("Hello, " + name + "!"); // Output: Hello, Alice!
Declaring variables

In JavaScript, variables are like labeled boxes that hold data you can use in your code. Here's how to create and use them:

Use let let or const keywords followed by a descriptive name (e.g., let age = 30). let allows you to change the variable's value later, while const keeps it constant. Once declared, refer to the variables's name to access its stored data (e.g., console.log("My age is: " + age);).

Variable scopes

When you declare a variable outside of any function, it is called a global variable, because it is available to any other code in the current document. When you declare a variable within a function, it is called a local variable, because it is available only within that function.

Example

let name = "Bob"; // String variable const PI = 3.14; // Constant variable (unchanging value) let total = 5 + 10; // Stores the result of an operation,

Data types
  • Number: Represents numeric values, including integers (whole numbers) and decimals (floating-point numbers). let age = 30; // Integer let pi = 3.14159; // Floating-point number
  • String: Represents textual data enclosed in quotes (single or double). let name = "Alice"; // String with text let message = 'This is a message with single quotes'; // String with single quotes
  • Boolean: Represents logical values: true or false Used for true/false conditions. let gameOver = true; // Boolean value (true) let isSunny = false; // Boolean value (false)
  • Undefined: Represents a variable that has been declared but not yet assigned a value. let unknown; // Variable declared without a value (undefined)
  • Null: Represents the intentional absence of any object value. It's different from undefined. let emptyBox = null; // Variable assigned the null value (intentionally empty)
  • Symbol: A unique and immutable (unchanging) identifier. Used less frequently, often for advanced object properties. let uniqueSymbol = Symbol("customSymbol"); // Creates a unique symbol
  • Object: A complex data structure that can hold collections of key-value pairs. We'll explore this further in future lessons!

Although these data types are a relatively small amount, they enable you to perform useful functions with your applications. Objects and functions are the other fundamental elements in the language. You can think of objects as named containers for values, and functions as procedures that your application can perform.

if..else Statement

Making Decisions with if...else in JavaScript Imagine your program needs to make choices based on certain conditions. That's where if...else statements come in JavaScript!

The Basic Idea:

  • You set a condition (like checking if a number is greater than 10).
  • If the condition is true, a block of code executes.
  • If the condition is false, an optional else block can run different code.

Example:

let age = 25; if (age >= 18) { console.log("You are eligible to vote!"); } else { console.log("Sorry, you are too young to vote."); }

Nesting if...else statements:

You can have multiple conditions within an if statement:

let grade = "A"; if (grade === "A") { console.log("Excellent work!"); } else if (grade === "B") { console.log("Great job!"); } else { console.log("Keep practicing!"); }
While loops

Imagine you have a task to repeat as long as a certain condition is met. That's what while loops are for in JavaScript!

How it works:

  • You set a condition (like checking if a counter variable is less than 5).
  • As long as the condition is true, a block of code executes repeatedly.
  • Inside the loop, you typically update the condition variable (like incrementing a counter) to eventually make the condition false and stop the loop.

Example:

let count = 1; while (count <= 5) { console.log("Count is: " + count); count++; // Update the counter to eventually stop the loop }
Function declarations

A function definition (also called a function declaration, or function statement) consists of the function keyword, followed by:

  • The name of the function.
  • A list of arguments to the function, enclosed in parentheses and separated by commas.
  • The JavaScript statements that define the function, enclosed in curly brackets{}.

Simple Function - Greetings

function greet(name) { console.log("Hello, " + name + "!"); } greet("Alice"); // Output: Hello, Alice!

Function with Calculations:

function calculateArea(length, width) { const area = length * width; return area; } const result = calculateArea(5, 10); console.log("The area is: " + result); // Output: The area is: 50

Function with Default Parameter:

function greetUser(name = "Guest") { // "Guest" is the default value console.log("Welcome, " + name + "!"); } greetUser(); // Output: Welcome, Guest! (Uses default name) greetUser("Bob"); // Output: Welcome, Bob!
